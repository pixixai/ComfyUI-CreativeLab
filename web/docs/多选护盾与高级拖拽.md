# Creative Lab (CLab) 核心架构：多选护盾与高级拖拽引擎

本文档深度解析 CLab 插件中关于卡片/模块的多选保护机制、批量删除逻辑，以及首创的“平行空间同步”拖拽算法和二维网格连选。

## 一、 系统级防误触护盾 (Global Selection Shield)

### 1. 痛点：被暗杀的多选状态

在浏览器标准的事件流中，鼠标交互的触发顺序永远是：
`mousedown` (按下) -> `mousemove` (移动) -> `dragstart` (起飞) -> `mouseup` (松开) -> `click` (点击完成)。

过去，当用户圈选了 3 个模块，准备按住其中一个进行拖拽时：

- 鼠标刚刚按下 (`mousedown`)，底层的“单选/取消多选”逻辑瞬间触发，判定用户没有按 `Ctrl` 键，于是无情地清空了多选数组。
- 等到 `dragstart` 触发时，多选阵列早已溃散，最终只能拖动单个模块。

### 2. 终极解法：Window 捕获级拦截器

我们在 `events/event_global.js` 中引入了最高权重的防误触护盾 `window._clabGlobalSelectionShield`。它完全模拟了 Windows/macOS 桌面系统的交互逻辑：

1. **事件劫持 (Capture Phase)**：护盾挂载在 `window` 对象的捕获阶段（`true`），确保它比任何组件都先拿到鼠标事件。
2. **状态侦测**：当 `mousedown` 发生时，护盾会检查被点击的元素是否**已经处于选中状态**（`state.selectedAreaIds.includes(id)`）。
3. **时停阻断**：如果已选中且未按修饰键，护盾立即调用 `e.stopPropagation()`，**掐断事件向下冒泡**。此时底层系统完全不知道用户按下了鼠标，多选状态得以完美保全！
4. **智能放行**：
    - 如果随后发生了 `dragstart`，多选阵列完美起飞。
    - 如果用户松开鼠标 (`mouseup`) 时发现并没有拖拽动作，护盾会主动代偿，执行“清空其他，仅保留当前”的单选逻辑。

## 二、 批量删除机制 (Batch Deletion)

卡片和模块右上角的 `✖` 删除按钮，采用了一套极其简洁但智能的判定逻辑：

1. **点击判定**：获取被点击的 `id`，检查它是否在多选数组（`state.selectedCardIds` 或 `state.selectedAreaIds`）中。
2. **分支执行**：
    - **批量模式**：如果被点击的项是高亮多选阵列的一员（且多选数量 > 1），系统会遍历整个 `state.cards`，将多选数组中的所有 `id` 对应的实体全部剔除。
    - **单点模式**：如果被点击的项并没有被选中，或者当前只是单选，系统仅精准摘除该 `id` 对应的单个实体。
3. **状态清理**：删除完成后，同步将这些 `id` 从内存的选中状态数组中抹除。

## 三、 模块级：智能双模拖拽引擎

我们在 `comp_modulearea.js` 中重写了 HTML5 Drag & Drop 引擎。当用户拖拽多个模块降落 (`drop`) 时，引擎会根据**“源卡片”**与**“目标卡片”**的关系，自动切换两种不同的排版魔法。

### 核心预热：起飞阶段 (Drag Start)

当拖拽发生时，引擎会将所有被选中的模块 ID 打包存入 `dragState.areaIds`。同时，扫描整个面板，在 `dragState.sourceInfo` 中精确记录每一个模块的**“原属卡片 ID”**和**“在数组中的原始索引 (Index)”**。

### 模式 A：平行空间同步位移 (同卡片拖拽)

**场景**：用户在 3 个不同的卡片中各选中了 1 个模块。用户拖住卡片 A 中的模块，在卡片 A 内部向下移动了 2 格。
**算法运作**：

1. 引擎发现鼠标松开的 `targetCardId` 等于拖拽主力（锚点）的 `dragState.cardId`。
2. 计算相对位移（Delta）：`Delta = 目标位置 Index - 锚点原始 Index`（比如向下两格，Delta = +2）。
3. 引擎遍历所有被选中的模块，不仅将卡片 A 中的模块移动到新位置，还会给卡片 B、卡片 C 中的模块施加同样的 `Delta` 位移。
4. **越界保护**：在应用 `newIdx = originalIdx + Delta` 时，强制使用 `Math.max(0, Math.min(newIdx, array.length))` 防止模块飞出卡片边界。
**视觉效果**：跨卡片的选中模块如同发生“量子纠缠”，在各自的卡片里平行、同步地上下跳跃。

### 模式 B：万物归一汇聚引擎 (跨卡片拖拽)

**场景**：用户依然选中了那 3 个模块。但这次，用户把鼠标拖出了卡片 A，直接扔到了卡片 D 里。
**算法运作**：

1. 引擎发现 `targetCardId` **不等于** `dragState.cardId`，判定为“跨界事件”。
2. 引擎瞬间开启“磁铁模式”，不再计算相对位移。
3. 将全宇宙（所有卡片）中被选中的这 3 个模块全部连根拔起（`splice`）。
4. 将它们打包为一个数组，一次性插入到目标卡片 D 的鼠标落点缝隙中。
**视觉效果**：散落在各地的模块瞬间跨越空间，全部聚合到了用户的鼠标位置。

## 四、 卡片级：阵列平移引擎

卡片的批量拖拽 (`comp_taskcard.js`) 逻辑相对线性，但同样强大：

1. **多选列队**：拖拽开始时，识别 `state.selectedCardIds`。
2. **中线判定**：在 `dragover` 阶段，计算目标卡片几何中心（`midX`）。鼠标在左侧，判定为 `dropPosition = 'left'`，亮起左侧绿边；反之亮起右侧。
3. **整体剥离与重组**：在 `drop` 时，新建 `remainingCards` 和 `movedCards` 两个数组。将选中的卡片从原队列抽离，然后在目标位置的索引处，利用 ES6 扩展运算符 `...movedCards` 将整个阵列一次性无缝插入。

## 五、 D矩阵框选 (二维网格连选) 引擎

在桌面操作系统的文件管理器中，按住 `Shift` 键点击可以进行范围连选。而在 CLab 中，因为模块分布在“多张卡片（X轴）”和“卡片内部列表（Y轴）”中，我们实现了一套**二维空间的矩阵连选算法**。

**场景**：用户先点击了卡片 1 的第 2 个模块（起点锚点），然后按住 `Shift` 键，点击了卡片 3 的第 4 个模块（终点）。

**算法运作 (`ui_panel.js` 的 `mousedown` 监听器中)**：

1. **锚点记录**：每次普通的单击，系统都会将该模块的 ID 记录为 `appState.lastClickedAreaId`。
2. **定位坐标**：当 `Shift + 点击` 发生时，系统通过遍历 `state.cards` 数组，精确找出起点和终点所在的 `cardIndex`（列号）和 `areaIndex`（行号）。
3. **建立边界框 (Bounding Box)**：
    - 最小列号 (`minC`) = `Math.min(起点卡片索引, 终点卡片索引)`
    - 最大列号 (`maxC`) = `Math.max(起点卡片索引, 终点卡片索引)`
    - 最小行号 (`minA`) = `Math.min(起点模块索引, 终点模块索引)`
    - 最大行号 (`maxA`) = `Math.max(起点模块索引, 终点模块索引)`
4. **矩阵扫描与捕获**：
通过双重 `for` 循环，遍历从 `minC` 到 `maxC` 的所有卡片，并在每张卡片中遍历从 `minA` 到 `maxA` 的所有模块。
5. **同类隔离**：为了防止输入模块和输出模块被混选导致工具栏冲突，扫描时会严格核对类型（`anchorType`），仅将类型相同的模块加入最终的 `state.selectedAreaIds` 数组中。
