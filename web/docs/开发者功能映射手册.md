## 一、 顶部全局工具栏与宏观排版

负责全局任务与工作流的管理操作。修改此处逻辑可能会引起全局重绘或内存指针变动。

- **新建任务 (Add Task Card)**
    
    ```
    1. `ui_panel.js`          (入口层：提供顶部工具栏 DOM 骨架与全局点击护盾)
    2. `comp_toolbar.js`      (触发层：绑定 [+ 新建任务] 事件，计算精准插入索引与焦点转移)
    3. `ui_state.js`          (状态层：向 state.cards 压入新数据，分配唯一 uuid)
    4. `comp_taskcard.js`     (排版层：生成 HTML，执行无延迟数学排版 updateCardsLayout，控制水平溢出滚动)
    5. `ui_selection.js`      (交互层：强制高亮新卡片并清理旧的选中状态)
    6. `state_manager.js`     (同步层：拦截状态突变，序列化全盘数据)
    7. `config_node.py`       (后端层：接收 JSON 字符串并作为 CLab_SystemConfig 图谱元数据永久落盘)
    8. `event_global.js`      (事件层：全局防误触护盾重新将新卡片纳入保护名单)
    ```
    
- **新建模块 (Add Module - 无界阵列并发)**
    
    ```
    1. `ui_panel.js`          (入口层：提供工具栏 DOM 骨架)
    2. `comp_toolbar.js`      (算法层：倒序(Descending)索引算法，支持跨卡片、多模块并发插入防偏移)
    3. `ui_state.js`          (状态层：静默保存，阻止全局核弹级重绘)
    4. `comp_modulearea.js`   (DOM操作：绕过 innerHTML，使用 appendChild/insertBefore 物理拼贴新节点)
    5. `module_input.js`      (渲染层：默认新建的是输入模块，提供无 widgetDef 时的基础骨架)
    6. `ui_utils.js`          (渲染层：为新模块注入自定义下拉框与事件监听)
    7. `state_manager.js`     (同步层：新模块数据持久化至工作流)
    8. `event_global.js`      (兜底层：确保新生成的 HTML 能正确响应 Esc 键与键盘左右切换)
    ```
    

## 二、 数据 I/O 与文件系统

极高风险区域，涉及前端沙盒突破、网络请求与 Python 文件读写。

- **导入 JSON (剪切板 / 本地文件)**
    
    ```
    1. `action_data_io.js`    (解析层：核心 FileReader 读取、剪切板 API 拦截与 JSON 容错解析)
    2. `ui_state.js`          (状态层：清空旧数据或执行 smart_append 智能追加)
    3. `comp_taskcard.js`     (渲染层：将海量卡片转化为 DocumentFragment 执行批处理插入，防 UI 卡死)
    4. `comp_modulearea.js`   (渲染层：深度渲染嵌套在卡片内部的表单，绑定物理拖拽)
    5. `ui_panel.js`          (视图层：触发重新计算滚动条边界，修正宽度滑块拉伸比例)
    6. `module_media.js`      (媒体层：如果导入了带视频的配置，唤醒媒体保险库准备装载)
    7. `event_global.js`      (护盾层：如果导入的 URL 触发 404，全局 error 拦截器立即铺设失效底图)
    8. `state_manager.js`     (同步层：立刻向后端图谱同步，防止 F5 刷新后数据丢失)
    ```
    
- **收集整理 / 后端物理迁移**
    
    ```
    1. `action_data_io.js`    (请求层：遍历提取带有输出结果的模块，构建文件路径与目标文件夹映射表)
    2. `api.py`               (后端层：核心安全区域！使用 os.commonpath 防御目录穿越，执行 shutil.move/copy)
    3. `ui_state.js`          (状态层：接口返回 success 后，在前端内存里强行重写历史记录 URL)
    4. `comp_modulearea.js`   (刷新层：触发外科手术 `window._clabSurgicallyUpdateArea`，无感替换媒体标签的 src)
    5. `module_output.js`     (渲染层：刷新网格视图中四宫格缩略图的路径)
    6. `api_injector.js`      (探针层：路径变更后触发 `loadeddata`，重新探测图片/视频的自然分辨率)
    ```
    

## 三、 执行引擎与 API 劫持

整个扩展中最复杂、技术密度最高的区域，与 ComfyUI 原生队列深度缠绕。

- **运行 / 循环运行全部 (Batch Run & Pruning)**
    
    ```
    1. `comp_toolbar.js`      (触发层：提取选中项，读取循环次数，调用执行器)
    2. `action_run_executor.js` (编译层：`buildTasksQueue`，使用展开运算符 `{...task}` 断开对象引用，拼接宏伟队列)
    3. `api_injector.js`      (劫持层 A：重写 `window.CLab.executeTasks`，接管 `app.queuePrompt` 进行排队)
    4. `api_injector.js`      (劫持层 B：重写 `app.graphToPrompt`，动态将面板 value 注入后端 payload)
    5. `api_injector.js`      (劫持层 C：**反向动态剪枝算法**，遍历节点依赖树，剔除与本次输出无关的冗余节点)
    6. `event_execution.js`   (反馈层：拦截 WebSocket 的 `progress_state` 与 `executing`，驱动蓝绿进度条)
    7. `event_execution.js`   (阻断层：捕获 `execution_error`，红灯报警并视 `_clabHaltOnError` 决定是否清空队列)
    8. `api.py`               (后端挂载：运行完成瞬间，执行 `/clab/copy_temp_asset` 截胡转移临时文件)
    9. `api_injector.js`      (闭环层：接收物理路径，覆盖前端内存，并调度 `_clabSurgicallyUpdateArea` 进行无感换源)
    ```
    

## 四、 动态工具栏与格式刷魔法

涉及高频的状态提取与克隆，对 `value` 和 `history` 的处理极其敏感。

- **克隆 (100% 完美镜像)**
    
    ```
    1. `action_module_config.js` (核心：执行深度 JSON 拷贝，保留 value、history 和历史封面索引，重置 UUID)
    2. `comp_modulearea.js`      (插入层：使用 `nextSibling` 计算物理落点，碎片化插入防屏闪)
    3. `comp_taskcard.js`        (排版层：如果是克隆卡片，同步调用 CSS 变量布局引擎撑开容器)
    4. `ui_state.js`             (保存层：落盘至 ComfyUI 图谱)
    ```
    
- **格式刷 (智能继承系统)**
    
    ```
    1. `action_module_config.js` (提取层：深拷贝源对象，**剥离所有历史图像**，但**绝对保留**输入模块的 value)
    2. `ui_panel.js`             (交互层：拦截 mousedown，识别目标是“覆盖”还是“新建”)
    3. `ui_panel.js`             (继承算法：覆盖时，比较新老坑位。目标已有的坑位拒绝覆写原 value；溢出的新坑位全盘继承格式刷 value)
    4. `comp_modulearea.js`      (刷新层：对目标区域执行 `window._clabSurgicallyUpdateArea` 局部更新)
    5. `ui_selection.js`         (善后层：退出 painterMode，鼠标恢复指针形状，移除面板高亮边框)
    ```
    
- **跨模块参数同步 (点穴爆破)**
    
    ```
    1. `action_batch_sync.js`    (算法层：`execSyncParams`，通过类型和 Title 标识匹配全图谱的同源模块)
    2. `ui_state.js`             (状态层：批量覆写 targetNodeId, targetWidget, matchMedia 等配置)
    3. `comp_modulearea.js`      (刷新层：对每一个被波及的模块独立调用微创更新，保持各自的视频进度条不中断)
    4. `module_input.js`         (渲染层：下拉框与 UI 根据新继承的类型重新渲染)
    ```
    

## 五、 媒体播放与视图组件

底层媒体组件，直接关乎内存表现和渲染帧率。

- **视频/音频保险库与生命周期**
    
    ```
    1. `ui_panel.js`          (金库调度：`window.CLab.stashMedia` 与 `restoreMedia`，拦截每次大重绘)
    2. `comp_modulearea.js`   (局部金库：`surgicallyUpdateArea` 中精准比对 `&t=` 时间戳，决定是恢复旧视频还是加载新视频)
    3. `module_media.js`      (分发层：根据文件名正则，挂载 `<audio>` 或 `<video>` 组件)
    4. `media_video.js`       (控制层：高精度防抖滚轮寻道、防冒泡音量滑杆、屏蔽默认进度条拖拽冲突)
    5. `media_audio.js`       (控制层：统一的 requestAnimationFrame 轮询器，突破父级画幅限制)
    6. `module_output.js`     (网格层：缩略图强挂 `preload="metadata"` 及 `#t=0.1` 截胡机制，防止爆显存)
    7. `event_global.js`      (底座层：兜底 404 错误，渲染 `#1e1e1e` 统一失效提示框)
    ```
    

## 六、 高级事件总线与拖拽引擎

管理全局用户输入的最高级中枢，确保交互绝对顺滑。

- **系统级多选与防误触护盾**
    
    ```
    1. `event_global.js`      (核心层：在 window `capture` 阶段截获 `mousedown` 与 `click`，识别修饰键)
    2. `ui_panel.js`          (判断层：白名单验证，放行 `INPUT`, `BUTTON`, `.clab-video-controls-interactive` 等控件)
    3. `ui_panel.js`          (矩阵算法：执行 `Shift + 点击` 的二维网格连选，计算边界框 Min/Max 坐标)
    4. `ui_selection.js`      (渲染层：剥离 DOM 刷新，仅对受影响节点的 `classList` 执行增删，实现零卡顿高亮)
    5. `action_batch_sync.js` (执行层：提供“删除相同模块”、“选择相同模块”的批量高亮切换逻辑)
    ```
    
- **双模物理拖拽引擎 (DnD)**
    
    ```
    1. `comp_taskcard.js`     (卡片层：执行中线吸附判定，提取拖拽体，利用 ES6 展开运算符实现阵列队列插队)
    2. `comp_modulearea.js`   (模块层：基于 `dragState` 分析目标卡片异同)
    3. `comp_modulearea.js`   (模式 A：同卡片触发“平行空间同步引擎”，计算 Delta 位移，带动全盘模块同步跳跃)
    4. `comp_modulearea.js`   (模式 B：跨卡片触发“磁铁汇聚引擎”，从全图谱剥离模块，瞬间扎堆插入目标落点)
    5. `comp_modulearea.js`   (善后层：执行 `updateAreaDOMIdentity` 重写拖拽完毕后残留的旧卡片数据烙印)
    6. `comp_modulearea.js`   (纠偏层：执行 `updateAllDefaultTitles` 静默重排序号)
    ```

全方位接管插件的运行偏好与 UI 视觉美学。

- **原生设置面板与所见即所得注入**
    
    ```

    3. `ui_utils.js`          (基建层：`injectCSS` 全面废弃硬编码，采用 `var(--clab-theme-xxx)` 声明组件样式)
    4. `ui_selection.js`      (交互层：卡片/模块的局部高亮边框和发光阴影自动绑定主题 CSS 变量)
    5. `comp_taskcard.js`     (渲染层：底部运行进度条自动继承主题色与动态发光渲染)
    6. `action_data_io.js`    (IO 层：动态读取 `window._clabArchiveDir` 实现个性化物理文件归档)
    7. `event_execution.js`   (容错层：动态读取 `window._clabHaltOnError` 决定是否阻截执行队列)
    ```
