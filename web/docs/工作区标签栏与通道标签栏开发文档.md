# 工作区标签栏与通道标签栏开发文档

更新时间：2026-04-05
适用版本：`schemaVersion = 3`

## 1. 文档目标

本文档用于说明 CreativeLab 中两层标签体系的实现：

- 底部：`工作区标签栏`（Workspace Bar）
- 顶部工具栏下方：`通道标签栏`（Channel Bar）

重点覆盖数据结构、渲染流程、事件行为、持久化链路，以及与绑定/注入逻辑的关系，便于后续继续做“通道局部刷新优化”等二期迭代。

## 2. 关键文件与职责

- `web/components/ui_state.js`
  - 状态模型、迁移规范、工作区/通道切换入口。
- `web/components/comp_workspace.js`
  - 工作区标签栏渲染与事件（增删改、多选、拖拽排序、右键菜单、重命名）。
- `web/components/comp_channel.js`
  - 通道标签栏渲染与事件（增删改、多选、拖拽排序、右键菜单、重命名）。
- `web/ui_panel.js`
  - 面板结构挂载、渲染编排、通道栏样式覆盖。
- `web/components/ui_utils.js`
  - 工作区栏与整体 UI 基础样式。
- `web/state_manager.js`
  - 与 `CLab_SystemConfig` 节点的序列化/反序列化同步。
- `web/api_injector.js`
  - 运行前注入（读取 area 上的绑定字段）。

## 3. 面板结构（DOM）

`web/ui_panel.js` 中面板结构为：

1. 顶部工具栏 `#clab-toolbar-handle`
2. 通道栏容器 `#clab-channel-strip > #clab-channel-bar`
3. 卡片区 `#clab-cards-container`
4. 底部工作区栏 `.clab-panel-footer > #clab-workspace-bar`

渲染顺序（`performPanelRender`）：

1. `renderDynamicToolbar`
2. `renderChannelBar`
3. `renderCardsList`
4. `renderWorkspaceBar`

每次渲染前后会保存并恢复：

- 卡片容器横向滚动
- 每张卡片 body 的纵向滚动
- 媒体播放节点（通过 stash/restore media vault）

## 4. 数据模型（Schema v3）

`ui_state.js` 当前核心结构：

```js
state = {
  schemaVersion: 3,
  activeWorkspaceId,
  workspaces: [
    {
      id,
      name,
      cards,
      activeChannelId,
      channels: [
        {
          id,
          name,
          bindings: { [areaId]: { ...bindingFields } },
          toolbarState: {}
        }
      ]
    }
  ],

  // active workspace mirror
  cards,
  activeCardId,
  activeChannelId,
  channels,
  toolbarState
}
```

绑定字段白名单由 `CHANNEL_BINDING_KEYS` 统一定义（如 `targetNodeId`、`targetWidget`、`targetNodeIds`、`targetWidgets`、`dataType`、`ratio` 等）。

### 迁移规则

- 旧数据如果没有 `channels`，会自动补一个默认通道。
- 默认通道会从当前 `cards[].areas[]` 提取历史绑定，避免升级后丢失绑定。

## 5. 切换与同步机制

### 5.1 工作区切换

`applyWorkspaceToState(workspace)`：

- 把目标 workspace 映射为运行态 `state.cards/state.channels`。
- 选中其 `activeChannelId`。
- 将该通道的 `bindings` 回放到 `cards[].areas[]`。

### 5.2 通道切换

`applyChannelToState(channelId)`：

- 仅切当前工作区下的 `activeChannelId`。
- 更新 `state.toolbarState`。
- 把目标通道 `bindings` 应用到当前 `state.cards` 的各 area。

### 5.3 写回时机

`syncStateToActiveWorkspace()` 在保存前执行：

- 把当前 `state.cards` 写回当前 workspace。
- 将当前卡片上的绑定抽取并写回“当前活动通道”的 `channel.bindings`。

这保证了“UI 上直接改 area 绑定字段”最终仍落到当前通道。

## 6. 工作区标签栏行为

由 `comp_workspace.js` 提供：

- 渲染：`renderWorkspaceBar`
- 事件：`attachWorkspaceEvents`
- 支持能力：
  - 单击切换
  - `Ctrl/Cmd` 多选
  - `Shift` 区间多选
  - 右键菜单（新建/复制/重命名/删除）
  - 双击重命名
  - 拖拽排序
  - 滚轮横向滚动

实现细节：

- 有“稳定 DOM 优化”：当 tab 结构未变时只更新 class/text，不整体重建，提升双击重命名稳定性。

## 7. 通道标签栏行为

由 `comp_channel.js` 提供：

- 渲染：`renderChannelBar`
- 事件：`attachChannelEvents`
- 支持能力：
  - 单击切换通道
  - `Ctrl/Cmd` 多选
  - `Shift` 区间多选
  - 右键菜单（新建/复制/重命名/删除）
  - 双击重命名
  - 拖拽排序
  - 滚轮横向滚动

当前通道切换路径仍为 `saveAndRender()` 全量渲染，不是“仅局部刷新绑定 UI”。

## 8. 样式与视觉规范

### 8.1 工作区栏样式来源

- `web/components/ui_utils.js`（`injectCSS`）

关键点：

- 非活动标签文字：`rgb(100, 100, 100)`
- 活动/选中背景：`rgb(24, 24, 24)`
- 末尾 `+` 按钮：`position: sticky; right: 0; z-index: 15; background: #181818`

### 8.2 通道栏样式来源

- `web/ui_panel.js`（`setupUI` 中的 `overrideStyle`）

关键点：

- 非活动标签文字：`rgb(100, 100, 100)`
- 活动/选中背景：`rgb(24, 24, 24)`
- 末尾 `+` 按钮：纯色不透明背景 `rgb(24, 24, 24)` + `opacity: 1`

### 8.3 相关布局项（与标签栏联动）

- 缩放滑杆 `#clab-card-width-ctrl`：`bottom: 44px`
- 卡片内容左右对称内边距：`10px`
- 卡片 body 使用 `overflow-y: overlay`，滚动条覆盖在右侧边距区域

## 9. 与绑定/执行链路的关系

通道隔离目前依赖“切换时回放绑定 + 保存前抽取绑定”的模式，而不是改造所有 action 的数据写口。

当前仍直接读写 `area.targetNodeId/targetWidget...` 的主要文件：

- `action_module_config.js`
- `action_binding.js`
- `action_batch_sync.js`
- `module_input.js`
- `api_injector.js`

为什么仍可工作：

1. 通道切换时：`channel.bindings -> area`（回放）
2. 用户编辑后保存时：`area -> channel.bindings`（抽取）

因此运行注入读取 area 字段时，会反映“当前激活通道”的结果。

## 10. 持久化与恢复

- 保存入口：`saveAndRender()` -> `window.CLab.saveState(state)`
- 节点写入：`StateManager.syncToNode()`（写 `scenes_data`）
- 反序列化：`StateManager.loadFromNode()` -> `hydrateStateFromPersisted()` -> `ensureWorkspaceRuntime()`

兼容性：

- 旧格式（无 `workspaces/channels`）可加载并迁移。

## 11. 已知现状与后续建议

- `channel.toolbarState` 已入模并随通道切换同步到 `state.toolbarState`，但当前尚未见独立消费逻辑，可视为预留字段。
- 通道切换仍为全量渲染路径；后续可改为“绑定区域局部刷新”，减少大面板重绘。
- JSON 导入导出（`action_data_io.js`）当前主要针对卡片/模块数据，不是 workspace+channel 全量结构导出入口。

## 12. 回归测试清单（建议每次改动后执行）

1. 工作区：新建/复制/重命名/删除/拖拽排序。
2. 通道：新建/复制/重命名/删除/拖拽排序。
3. 通道切换后绑定是否隔离（A 通道改绑定，不污染 B）。
4. 保存后重开工作流，工作区与通道是否完整恢复。
5. 运行注入是否读取当前通道绑定。
6. `+` 按钮是否保持不透明，标签颜色是否符合规范。
7. 缩放滑杆位置与卡片左右边距是否符合当前设计。

## 13. 开发约定（新增字段/能力时）

新增“通道级绑定字段”时，至少同步修改：

1. `CHANNEL_BINDING_KEYS`
2. `extractAreaBinding()`
3. `resetAreaBindingFields()`
4. `applyAreaBinding()`

新增“标签栏交互能力”时，工作区与通道应尽量保持对称实现（避免两套行为分叉）。
