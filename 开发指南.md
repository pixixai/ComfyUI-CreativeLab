# ComfyUI ShellLink 开发者指南

本文档面向对 ShellLink 插件进行二次开发或维护的工程师，详述了组件化重构后的文件职责、系统架构以及核心函数的调用说明。

## 1. 架构概述

ShellLink 采用了**完全解耦的组件化前端架构**。数据层 (Model)、视图层 (View) 和拦截层 (Controller) 被严格分离：

- **数据流向**：视图组件修改 `ui_state.js` 中的中心化数据 -> 调用 `saveAndRender()` -> 触发 DOM 局部重绘 -> 将 JSON 序列化并存入 ComfyUI 画布上的 `ShellLinkSystemConfig` 节点。
- **执行拦截**：`api_injector.js` 劫持 `app.queuePrompt`。在向 Python 后端发送请求前，从 state 中读取参数，深拷贝并修改 Node 的 inputs，最后投递执行。

## 3. 核心文件详解与函数清单

### `web/main.js`

插件的注册入口，统筹各模块的初始化。

- `setup()`: 扩展加载入口，调用 UI 初始化与 API 劫持。包含清空图谱时重置面板的钩子。
- `loadedGraphNode(node)`: 当从本地加载包含配置节点的工作流时，触发数据恢复。
- `nodeCreated(node)`: 监听配置节点的创建或粘贴事件，实现跨工作流复制支持。

### `web/api_injector.js`

整个插件最核心的“偷天换日”逻辑，负责将 UI 参数注入底层执行流。

- `setupAPIInjector(app)`: 挂载 API 劫持与全局函数。
- `window.ShellLink.runCard(cardId)`: 执行单个任务卡片的快捷方法。
- `window.ShellLink.executeTasks(tasks)`: 核心调度方法。根据传入的任务队列，对 Prompt 字典进行动态剪枝（隔离无用节点）、参数替换（注入 Input 模块值）、以及路径重写（变 PreviewImage 为 ShellLinkSaveImage 并修改存图前缀），最后触发 `app.api.queuePrompt`。

### `web/state_manager.js`

管理前端内存与 ComfyUI Canvas 物理节点的数据同步。

- `startWatchdog(graph)`: 轮询看门狗，侦测外部引起的节点数据变化（如撤销、粘贴）。
- `getConfigNode(graph)`: 在当前画布中查找最有效的 `ShellLinkSystemConfig` 节点。
- `syncToNode(graph)`: 将 `ui_state` 的数据序列化写入画布节点。
- `loadFromNode(graph)`: 从画布节点读取 JSON 并派发 `shell_link_state_loaded` 事件。
- `createConfigNode(graph)`: 辅助方法，在画布上自动生成配置锚点节点。

### `web/ui_panel.js`

主面板容器，负责组装组件并深度监听系统事件。

- `setupUI()`: 构建 DOM，注入 CSS，并在 ComfyUI 侧边栏注册启动入口。
- `togglePanel()`: 呼出/隐藏主面板。
- `performRender()`: 核心渲染调度器，调用子组件进行更新。
- `setupGlobalEventListeners()`: 绑定快捷键，拦截 WebSocket 的 `progress`、`execution_start` 和 `execution_error` 以驱动进度条和报错提示。
- `enterBindingModeForSelected(targetType)`: 开启沉浸式拾取模式，拦截 `pointerup` 事件以拾取画布节点。

### `web/components/ui_state.js`

纯前端状态仓库。

- `state`: 存储卡片队列、选中状态、格式刷状态等。
- `dragState`: 存储正在进行 DnD 拖拽的元素标识。
- `saveAndRender()`: 被所有 UI 交互调用，持久化状态并派发 `sl_render_ui` 重绘事件。

### `web/components/ui_utils.js`

通用工具与样式库。

- `buildCustomSelect(...)`: 生成具有标准 UI 风格的自定义下拉菜单 HTML。
- `getWidgetDef(nodeId, widgetName)`: 遍历 ComfyUI 内存图谱，获取目标参数类型的定义。
- `getCustomNodeMenuHTML(...)`: 解析并渲染带有层级结构（Group）的节点选项菜单。
- `showBindingToast(msg, isError)` / `hideBindingToast()`: 控制顶部沉浸式提示条。
- `injectCSS()`: 将独立作用域的 CSS 样式注入到 Document 中。

### `web/components/comp_toolbar.js`

管理静态与动态工具栏。

- `setupStaticToolbarEvents(container)`: 绑定顶部全局工具栏事件（新建、导入导出、运行队列组装）。
- `renderDynamicToolbar(container)`: 根据当前选中的模块类型（Input/Output）和数量，动态渲染出对应的工具参数 HTML（含批量操作菜单）。
- `attachDynamicToolbarEvents(container)`: 为动态工具栏中的输入、下拉、克隆、格式刷、批量同步、位移等功能绑定交互事件。

### `web/components/comp_taskcard.js`

任务卡片的骨架级渲染与交互。

- `renderCardsList(container)`: 渲染卡片列表，计算溢出以决定是否进行 Flex 居中，并将“新建任务”剥离出居中流。
- `attachCardEvents(container)`: 绑定卡片标题修改、删除、卡片多选点击、以及基于 HTML5 DnD 的**卡片级拖拽重排**与跨卡片**模块接收**逻辑。

### `web/components/comp_modulearea.js`

底层模块的精细渲染与交互反馈。

- `generateAreaHTML(area, card)`: 输出模块内的所有 HTML 元素（包括输入框、下拉菜单、预览图片框架以及输出模块的进度条容器）。
- `attachAreaEvents(container)`: 负责模块内属性的变更。包含基于 `mousedown` 的零延迟热更新逻辑（只重绘工具栏，不刷新光标焦点），以及模块在当前卡片内的 DnD 拖拽排序逻辑。

### `nodes/config_node.py` (后端)

- `shell_link_organize_files(request)`: 异步 REST API 路由。接收前端的归档请求，利用 `shutil` 将 `output/` 目录下的图片移动或复制到指定的子文件夹，并执行重命名。
- `ShellLinkSystemConfig` (Class): 继承 ComfyUI 节点规范，开放一个隐藏的 `STRING` 端口，作为纯净的数据持久化仓库。
