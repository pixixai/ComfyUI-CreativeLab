## 顶部全局工具栏

位于面板最上方，负责全局任务与工作流的管理操作。

- 新建任务
    
    ```
    1. `ui_panel.js` (UI入口：提供顶部工具栏 DOM 骨架)
    2. `comp_toolbar.js` (事件层：绑定 [+ 新建任务] 按钮的点击事件)
    3. `comp_taskcard.js` (业务层：包含 `addNewCard` 核心逻辑，负责 push 新数据并滚动到底部)
    4. `ui_state.js` (状态层：维护 `state.cards` 数组并派发全局重绘事件)
    5. `state_manager.js` (同步层：拦截状态变化，准备向 ComfyUI 节点写入)
    6. `config_node.py` (后端层：接收并持久化保存 JSON 字符串)
    ```
    
- 新建模块
    
    ```
    1. `ui_panel.js` (UI入口：提供工具栏 DOM 骨架)
    2. `comp_toolbar.js` (业务层：判断当前 `state.activeCardId`，并在该卡片的 `areas` 数组末尾添加新模块结构)
    3. `ui_state.js` (状态层：执行 `saveAndRender()`)
    4. `comp_modulearea.js` (渲染层：被重绘引擎调用，生成新模块的 HTML)
    5. `module_input.js` (渲染层：因为默认新建的是输入模块，由该文件完成具体表单渲染)
    6. `state_manager.js` (同步层：将新加入的模块数据同步至图谱)
    ```
    

## 导入

从剪切板导入JSON数据

- 创建任务：
    
    ```
    1. `action_data_io.js` (核心：下拉菜单 DOM 注入，以及 `handleClipboardImport` 剪切板读取拦截)
    2. `action_data_io.js` (解析层：`processImportedJSON` 方法中的 `mode === 'new'` 逻辑分支，将 JSON 转化为 Card 对象)
    3. `ui_state.js` (状态层：写入转换后的多张卡片，并派发重绘)
    4. `ui_panel.js` (视图层：响应重绘事件，生成新的卡片容器)
    5. `comp_taskcard.js` (视图层：渲染具体的卡片 HTML)
    ```
    
- 追加模块：
    
    ```
    1. `action_data_io.js` (核心入口：`handleClipboardImport`)
    2. `action_data_io.js` (解析层：`processImportedJSON` 中的 `mode === 'smart_append'` 分支，计算索引并处理溢出新建卡片逻辑)
    3. `ui_state.js` (状态更新与重绘广播)
    4. `state_manager.js` (同步至 ComfyUI 节点树)
    ```
    
- 追加模块到选中：
    
    ```
    1. `action_data_io.js` (核心入口：`handleClipboardImport`)
    2. `action_data_io.js` (解析层：`processImportedJSON` 中的 `mode === 'append_selected'` 分支，校验多选数组 `state.selectedCardIds`)
    3. `ui_selection.js` (渲染层：导入完成后，维持这些卡片的选中高亮状态)
    ```
    

从本地导入JSON数据

- 创建任务：
    
    ```
    1. `action_data_io.js` (核心引擎：`handleLocalImport`，负责创建虚拟 `<input type="file">`，调用 FileReader 读取本地文件)
    2. `action_data_io.js` (解析：将 FileReader 结果送入 `processImportedJSON` 的 `new` 模式)
    3. `ui_state.js` (状态持久化与重绘)
    ```
    
- 追加模块：
    
    ```
    1. `action_data_io.js` (入口：`handleLocalImport`)
    2. `action_data_io.js` (解析：送入 `smart_append` 模式处理)
    3. `ui_state.js` (重绘与保存)
    ```
    
- 追加模块到选中：
    
    ```
    1. `action_data_io.js` (入口：`handleLocalImport`)
    2. `action_data_io.js` (解析：送入 `append_selected` 模式处理)
    3. `ui_state.js` (重绘与保存)
    ```
    

## 运行

- 运行
    
    ```
    1. `comp_toolbar.js` (触发器：绑定 `sl-btn-run`，提取当前选中的单卡片或输出模块数据)
    2. `action_run_executor.js` / `api_injector.js` (执行队列管理器：`window.ShellLink.executeTasks`，将任务压入 `_slExecQueue`)
    3. `api_injector.js` (核心劫持器：拦截 `app.graphToPrompt`，将前端参数强行注入进发往后端的 JSON 包)
    4. `api_injector.js` (剪枝引擎：根据选中的输出节点，反向遍历剔除无关的 ComfyUI 节点，节省算力)
    5. `event_execution.js` / `api_injector.js` (进度反馈：监听 `progress` 等 WebSocket 事件，控制 UI 进度条与错误红灯)
    6. `api.py` (后端钩子：如果运行完成，触发 `copy_temp_asset` 接口将预览图转移至永久存储)
    ```
    
- 运行全部（下拉选项）
    
    ```
    1. `comp_toolbar.js` (触发器：遍历 `state.cards`，将所有卡片打包为批量任务数组)
    2. `api_injector.js` (排队引擎：将数组全部压入 `_slExecQueue`，并接管 `queuePrompt` 实现自动接力运行)
    3. `event_execution.js` (进度反馈：控制多卡片并行的进度条状态)
    ```
    

## 导出

打包为ZIP

- 下载全部
- 下载选中
- 下载全部（含所有生成记录）
- 下载选中（含所有生成记录）
    
    ```
    1. `action_data_io.js` (视图与逻辑入口：提供导出下拉菜单 DOM，并包含 `downloadMediaFiles` 方法)
    2. `action_data_io.js` (打包引擎：异步引入 `JSZip` 库，执行前端内存打包与 Blob 转换)
    3. `ui_state.js` (数据源：提供所有待遍历的 `state.cards` 及其 history 历史记录数组)
    4. `ui_utils.js` (UI 交互：调用 `showBindingToast` 显示“正在打包”的悬浮提示)
    ```
    

收集整理

- 移动到子文件夹
- 复制到子文件夹
    
    ```
    1. `action_data_io.js` (逻辑入口：`organizeOutputFiles`，遍历查找所有带媒体结果的模块，构建文件路径映射表)
    2. `api.py` (服务端文件操作层：处理 POST `/shell_link/organize_files`，执行 Python 的 `shutil.move` 或 `shutil.copy2`)
    3. `ui_state.js` (状态同步层：接口返回成功后，在前端内存里修改所有图片的 URL 路径，防止图片显示裂开)
    4. `ui_panel.js` / `comp_modulearea.js` (视图刷新层：重新渲染新路径的媒体文件)
    ```
    

导出JSON数据

- 输入模块
- 输出模块
- 全部模块
    
    ```
    1. `action_data_io.js` (逻辑入口：`handleJsonExport` 判断当前是复制还是下载模式)
    2. `action_data_io.js` (序列化引擎：`generateExportJSON` 根据不同模式过滤参数，格式化导出文本)
    3. `ui_state.js` (数据源提供者)
    ```

# 动态工具栏

卡片模块通用工具

- 克隆
    
    ```
    1. `action_module_config.js` (触发器：渲染克隆按钮并绑定 `#tb-clone-btn` 事件)
    2. `action_module_config.js` (逻辑层：判断深拷贝的目标是卡片还是模块，分配新 ID 并插入数组)
    3. `ui_state.js` (调度层：开启 500ms 护盾，执行静默保存 `saveAndRender(true)`)
    4. `comp_modulearea.js` (DOM 操作层：如果是模块克隆，直接将新 HTML 通过 `after()` 插入 DOM 树，防止全局闪烁)
    5. `ui_selection.js` (反馈层：更新新克隆对象的蓝/绿框高亮)
    ```
    
- 格式刷
    
    ```
    1. `action_module_config.js` (状态开关：切换 `state.painterMode` 开启/关闭，并缓存源数据到 `state.painterSource`)
    2. `ui_panel.js` (中央拦截器：`cardsContainer.addEventListener("click")` 劫持一切点击事件，判断是否在格式刷模式并执行覆盖/插入)
    3. `ui_state.js` (调度层：静默保存 `saveAndRender(true)`)
    4. `comp_modulearea.js` (DOM 操作层：如果是刷入新模块，执行 DOM 的精确定点插入)
    5. `ui_state.js` (DOM 替换层：如果是覆盖现有模块，调用 `surgicalUpdateArea` 执行外科手术替换)
    ```
    

模块工具

- 拾取节点
    
    ```
    1. `action_module_config.js` (触发器：点击准星按钮，派发 `sl_enter_binding_mode` 自定义事件)
    2. `action_binding.js` (核心拦截引擎：隐藏面板，接管 ComfyUI 画布的右键菜单与点击事件，反查鼠标位置对应的节点及连线)
    3. `ui_utils.js` (反馈层：弹出顶部的“请在工作流中点击...”提示 Toast)
    4. `ui_state.js` (更新绑定数据并恢复面板显示)
    ```
    
- 关联节点
    
    ```
    1. `action_module_config.js` (渲染层：构建节点下拉选择框骨架)
    2. `ui_utils.js` (数据层：`getCustomNodeMenuHTML` / `getMultiNodeMenuHTML`，遍历 ComfyUI 的 group 结构，生成按组分类的树状节点列表 HTML)
    3. `ui_state.js` (数据同步层：将选中的节点 ID 写入 state)
    ```
    
- 重置
    
    ```
    1. `action_module_config.js` (触发器：点击 `#tb-reset-module`)
    2. `action_module_config.js` (逻辑层：将该模块的 `targetNodeId`, `targetWidget`, `value` 等所有配置参数全部恢复默认)
    3. `ui_state.js` (调度层：全量重绘更新)
    ```
    
- 同步参数
    
    ```
    1. `action_module_config.js` (触发器：点击同步按钮)
    2. `action_batch_sync.js` (执行引擎：`execSyncParams`，遍历整个 `state.cards`，通过标题匹配寻找同类模块，执行深度克隆覆盖)
    3. `ui_state.js` (重绘与保存)
    ```
    

输入模块

- 绑定参数
    
    ```
    1. `action_module_config.js` (逻辑处理：处理下拉菜单选择结果)
    2. `ui_utils.js` (数据嗅探：`getWidgetDef` 深入 ComfyUI 内部抓取指定节点该参数的默认类型、类型要求或枚举列表)
    3. `module_input.js` (渲染器：根据嗅探到的类型，决定渲染文本框 (`textarea`)、开关 (`checkbox`)、下拉框 (`select`) 或文件上传区)
    4. `state_manager.js` (存储数据)
    ```
    
- 参数框高度适配
    
    ```
    1. `action_module_config.js` (状态控制：处理复选框的勾选，更新 `autoHeight` 布尔值)
    2. `module_input.js` (视图层：如果为 true，为其附加 `height: auto` 样式，并监听 `oninput` 自动设置 `scrollHeight`)
    ```
    

输出模块

- 模块比例 / 宽度 / 高度
    
    ```
    1. `action_module_config.js` (状态控制：收集下拉菜单预设与手工输入的 W 和 H 值)
    2. `ui_utils.js` (样式计算：`getRatioCSS` 将设定值转化为标准的 CSS `aspect-ratio` 属性)
    3. `module_output.js` (视图渲染：将计算好的比例 CSS 注入到 `.sl-preview-bg` 画框上)
    ```
    
- 显示模式
    
    ```
    1. `action_module_config.js` (状态控制：处理显示全部、填充、拉伸选项)
    2. `module_output.js` (解析层：将设定值转译为 `object-fit` 参数)
    3. `module_media.js` / `media_video.js` / `media_image.js` (渲染层：将 `object-fit: contain/cover/fill` 应用于底层的 `<video>` 或 `<img>` 标签)
    ```
    
- 匹配媒体比例
    
    ```
    1. `action_module_config.js` (触发开关：更新 `matchMedia` 状态)
    2. `api_injector.js` (拦截探针：利用隐藏的 `window._slMediaLoadHijacked`，截获图片或视频的原生 `load/loadeddata` 事件，反向读取其真实像素宽高)
    3. `ui_state.js` (状态修正：将探测到的真实比例强行覆写进 state 并触发局部自适应重绘)
    ```
    
- 管理生成记录
    
    ```
    1. `action_module_config.js` (触发器：切换 `isManageMode` 状态)
    2. `module_output.js` (渲染切换：根据状态判断，渲染为全屏媒体模式，或是包含四宫格缩略图的历史记录 `.sl-history-grid` 模式)
    3. `comp_contextmenu.js` (关联层：在网格模式下，限制右键菜单的行为表现)
    ```

    ```
    1. `comp_contextmenu.js` (入口：`sl-ctx-clear`)
    2. `comp_contextmenu.js` (逻辑层：将当前模块的 `area.history` 直接重置为空数组 `[]`)

    ```
    
- 清除所有并删除本地文件
    
    ```
    1. `comp_contextmenu.js` (入口：`sl-ctx-clear-del`)
    2. `ui_state.js` (`removeUrlsGlobally` 清除全局所有涉及这些历史记录的软链接，释放锁)
    3. `comp_contextmenu.js` (延时循环调用 `deletePhysicalFile`)
    4. `api.py` (执行底层的批量 `os.remove`)
    ```

# 模块操作

- 生成记录 (网格内的交互操作)
    
    ```
    1. `module_output.js` (视图层：生成网格容器与缩略图结构 `.sl-history-thumb`)

    ```

    ```
    3. `media_video.js` (交互控制：实现右键左右拖拽快进/倒退的 `isScrubbing` 特殊交互逻辑)

    7. `ui_panel.js` (白名单拦截：在其内部将 `.sl-video-controls-interactive` 等元素加入中央拦截器的放行名单，保障拖拽事件畅通)
    ```
