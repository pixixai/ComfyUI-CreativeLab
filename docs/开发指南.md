# ComfyUI ShellLink 开发者指南

本文档面向对 ShellLink 插件进行二次开发或维护的工程师，详述了经过**模块化极限重构**后的文件职责、系统架构以及核心函数的调用说明。

## 1. 架构概述

ShellLink 采用了完全解耦的组件化前端架构（MVC / 乐高模式）。数据层 (Model)、视图层 (View) 和控制交互层 (Controller / Actions / Events) 被严格分离，并引入了高性能的渲染保护机制：

- **数据流向**：视图组件修改 `ui_state.js` 中的中心化数据 -> 调用 `saveAndRender()` -> 触发事件总线 -> 执行 DOM 重绘 -> 将 JSON 序列化并存入 ComfyUI 画布上的 `ShellLinkSystemConfig` 节点。
- **职责彻底单一化**：
    - `components/` 下的根组件负责“长什么样”（DOM 渲染与基础组装）。
    - `actions/` 负责所有的业务计算与复杂交互（如导出导入、跨卡片同步、编译队列）。
    - `events/` 负责充当神经系统（监听 WebSocket、全局快捷键、局部拦截器）。
    - `modules/` 细化了具体的输入/输出参数表单和多媒体底层播放逻辑。
- **媒体保险库 (Media Vault) 防闪烁机制**：为了对抗 ComfyUI 高频的数据同步引起的 DOM 重建，系统在 `ui_panel.js` 钩入了“DOM拔插术”。在重绘前将播放中的音视频物理剥离至内存，重绘后精准插回，实现 0 卡顿的无缝重绘。
- **执行劫持引擎**：`api_injector.js` 劫持 `app.queuePrompt`。在发包前注入 UI 参数并根据输出需求对图谱进行动态剪枝。

## 3. 核心文件与函数 / 接口全映射

### 🖥️ 3.1 核心驱动层 (`web/`)

- **`main.js`**
    - `setup()`: 扩展加载入口，调用 UI 初始化与 API 劫持。
    - `loadedGraphNode()` / `nodeCreated()`: 监听配置节点事件，实现工作流载入时的数据恢复。
- **`state_manager.js`**
    - `syncToNode(graph)`: 将前端 `ui_state` 对象序列化为 JSON，写入 ComfyUI 画布隐藏节点的 STRING 端口。
    - `loadFromNode(graph)`: 反序列化节点数据并写入前端内存。
    - `startWatchdog(graph)`: 轮询看门狗，侦测因“撤销/重做”等导致的节点数据突变。
- **`api_injector.js`**
    - `setupAPIInjector()`: 挂载核心劫持环境。
    - `window.ShellLink.executeTasks()`: 推入多卡片排队队列。
    - `app.graphToPrompt (劫持)`: **核心拦截**。将输入模块参数深拷贝注入 Prompt，并基于输出模块目标执行**动态节点剪枝**。
    - `api.addEventListener("executed")`: 截胡归档引擎。拦截出图事件，直接调用 `/shell_link/copy_temp_asset` 接口，将 `temp` 或 `output` 的生成文件转移至专属归档目录。
- **`ui_panel.js`**
    - `setupUI()` / `createPanelDOM()`: 构筑 `absolute` 的主面板与遮罩，注入极简全局 CSS。
    - `cardsContainer.addEventListener("mousedown/click")`: **中央集权事件防火墙**。通过 `isInteractive` 与 `isMedia` 白名单拦截并管理卡片与模块的排斥选中、Ctrl 多选。
    - `ShellLink.stashMedia()` / `ShellLink.restoreMedia()`: 触发媒体保险库，执行 DOM 剥离与归还保护。

### 📦 3.2 基础组件与状态层 (`web/components/`)

- **`ui_state.js`**
    - `state` / `appState` / `dragState`: 唯一的数据信源。
    - `saveAndRender(silent)`: 核心调度。当 `silent=true` 时开启 500ms 护盾阻止核弹级重绘，配合局部替换实现丝滑更新。
    - `removeUrlsGlobally(urls)`: **资产清理锁**。遍历全图谱清除对文件的引用，迫使浏览器松开本地文件的读取锁（用于配合物理删除）。
- **`ui_selection.js`**
    - `updateSelectionUI()`: 外科手术级选中引擎。仅遍历更改 DOM 的 `classList` (边框变绿变蓝)，不触发任何 HTML 重绘。
- **`ui_utils.js`**
    - `getWidgetDef(nodeId, widget)`: 深入底层解析目标节点的参数类型。
    - `getCustomNodeMenuHTML()`: 将 ComfyUI 的节点数据按 Group 解析并渲染为层级树状 HTML。
    - `showBindingToast()`: 呼出沉浸式的全局悬浮提示。
- **`comp_contextmenu.js`**
    - `setupContextMenu()`: 右键菜单调度中心。
    - `deletePhysicalFile()`: 结合后端 API 彻底抹杀 `output` 文件。
    - 包含**移除软链接**与**彻底物理删除**两套独立隔离的核心逻辑分支。
- **`comp_modulearea.js` / `comp_taskcard.js`**
    - 分发构建具体模块的 HTML。包含 HTML5 `dragstart / dragover / drop` 的中线判定算法，实现卡片和模块的无感拖拽重排。

### ⚙️ 3.3 交互控制器层 (`web/components/actions/`)

- **`action_data_io.js`**
    - `processImportedJSON()`: 反序列化剪切板或本地 JSON 数据，支持 `new` (新建) 和 `smart_append` (智能追加) 模式。
    - `downloadMediaFiles()`: 调用 `JSZip` 挂载前端内存 ZIP 打包与批量下载。
    - `organizeOutputFiles()`: 收集历史记录并调用后端 `/shell_link/organize_files` 实现项目级目录迁移。
    - `uploadImageToServer()`: 发送 `FormData` 完成本地文件拖拽至服务器的转换。
- **`action_module_config.js`**
    - `renderDynamicToolbar()`: 根据模块类型生成极其复杂的**悬浮动态工具栏**。
    - 管理**格式刷 (Format Painter)** 的全流程拦截覆盖。
- **`action_binding.js`**
    - `enterBindingModeForSelected()`: 拦截原生画布，通过 `processContextMenu` 阻断器实现点击节点反查连线 (Links) 与插槽 (Slots) 的一键绑定映射。
- **`action_batch_sync.js`**
    - `execSyncParams()` / `execMoveForward()`: 高阶数组算法，实现卡片队列中的跨越查询、同名参数同步覆盖及批量整体平移。

### 📡 3.4 事件与生命周期 (`web/components/events/`)

- **`event_execution.js`**
    - `setUIProgress()`: 并发进度条引擎，支持蓝绿常态以及 Error 红灯警报模式。
    - **熔断拦截机制**: 捕获 `api.queuePrompt` 的底层 HTTP `error` 返回以及 `execution_error`。一旦报错，自动清空排队队列 `_slExecQueue`，防止卡死。
- **`event_global.js`**
    - `keydown` 监听器：处理 `ESC` 一键清空全选状态。
    - **丝滑切图引擎**: 监听 `ArrowLeft / ArrowRight`，直接调用 `renderMedia` 并定点覆盖 HTML，阻止重绘波及其他模块。

### 🎬 3.5 内部模块与多媒体底座 (`web/components/modules/`)

- **`module_input.js`**
    - 根据探测到的 Widget 定义，动态生成输入框、布尔开关 (Checkbox) 或文件拖拽上传区 (`sl-upload-zone`)。
    - 支持 `scrollHeight` 动态参数框高度适配。
- **`module_output.js`**
    - `isManageMode`: 网格视图管理引擎。支持 Ctrl/Shift 框选多张图片缩略图，并进行内部重新排序。
    - 包含右侧 `✖` 快速移除的轻量操作。
- **`module_media.js` & `media_types/*`**
    - **总路由**: 根据 `getMediaType` (探测 .mp4, .m4a, .png 等) 将渲染工作分发给对应组件。
    - `media_video.js`: 视频播放器实现。包含 `isScrubbing` (右键滑动预览进度)、防冒泡音量滑杆、原生画中画 API 唤起。
    - `media_audio.js`: 带有黑胶滤镜的专属音频播放器，接管 `<audio>` 元素的控制状态。
    - **全局 RAF 引擎**: 使用单例 `requestAnimationFrame` 轮询更新全版面所有音视频的进度条宽度与 Timecode。

### 🐍 3.6 后端 Python 环境 (`api.py`)

- **`class ShellLinkSaveImage(nodes.SaveImage)`**
    - 动态注入的自定义图片保存节点。彻底去除了官方节点讨厌的尾部下划线，并在内部实现了强大的 `while os.path.exists` 防覆盖自增补丁。
- **`@routes.post("/shell_link/delete_file")`**
    - 获取 `filename`, `subfolder` 及 `type` (区分 temp/output)，执行操作系统的 `os.remove()`。
- **`@routes.post("/shell_link/organize_files")`**
    - 使用 `shutil.move` 与 `shutil.copy2`，将生成杂乱的文件按工作流节点重构为完美的树状归档目录。
- **`@routes.post("/shell_link/copy_temp_asset")`**
    - **截胡专用接口**：瞬间将临时目录 (`temp`) 或默认输出目录 (`output`) 下的文件复制到 `ShellLink/video` 或 `ShellLink/audio` 等归档子文件夹中，以保障前端重启后媒体文件永不丢失。
