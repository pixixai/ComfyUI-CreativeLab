# ShellLink 局部微创更新重构记录

## 一、 重构背景：为什么要抛弃全量重绘？

**Q：原本的架构出了什么问题？**
A：在 ShellLink 的早期版本中，我们采用了极其经典的“数据驱动视图”模式。无论用户是修改了标题、点选了下拉框、还是输入了文本，最终都会调用 `saveAndRender()` 函数。
该函数的工作原理是：**把整个卡片区域的 `innerHTML` 直接全部推翻，然后根据最新的数据重新生成一遍 DOM 树。**

**Q：为什么这会导致视频闪烁或消失？**
A：对于静态的文字或图片来说，`innerHTML` 替换速度极快，肉眼看不出破绽。但对于原生的 HTML5 `<video>` 和 `<audio>` 标签来说，一旦它们的父容器被 `innerHTML` 销毁并重建，**旧的视频节点就会被彻底销毁，新的视频节点会被重新插入**。这会导致：

1. 浏览器重新建立媒体解码管线，出现明显的“黑屏”或“白屏”闪烁。
2. 正在播放的进度瞬间归零。
3. 如果操作过于频繁（例如一直按住方向键切换），会导致浏览器内存泄露和卡顿。

## 二、 核心解法：“微型物理金库”与定点爆破

为了解决闪烁问题，我们引入了 **外科手术级定点爆破 (Surgical Update)** 以及 **微型金库拦截 (Mini-Stash)** 技术。

### 1. 核心引擎：`surgicallyUpdateArea`

这个函数定义在 `comp_modulearea.js` 中。它的工作流程如下：

1. **定位病灶**：根据传入的 `areaId`，精准找到对应的那一块 HTML 模块节点。
2. **提取心脏 (Mini-Stash)**：在动手拆除 DOM 之前，先检查里面有没有 `<video>` 或 `<audio>` 标签。如果有，直接用原生 JS 把它“拔”出来，塞进页面底部一个隐藏的 `div#sl-mini-vault`（金库）中保管。此时，视频在后台依然保持缓冲或播放状态。
3. **定点替换**：仅重新生成当前这一个模块的 HTML 字符串，替换掉旧的节点，并重新绑定事件。
4. **物归原主**：新节点生成后，系统检查里面是否需要渲染媒体。如果需要，直接从“金库”里把刚才那个活着的 `<video>` 原封不动地贴回新 DOM 里，并且恢复其 `currentTime` 播放进度。

### 2. 避免循环依赖的“天降神兵”模式

因为几乎所有的 UI 模块都需要调用这个局部刷新函数，如果大家都在文件头 `import { surgicallyUpdateArea }`，会导致 ES6 模块产生**循环依赖 (Circular Dependency)**，整个扩展直接崩溃白屏。
**解决方式**：在 `comp_modulearea.js` 加载时，我们将该函数挂载到了全局对象上：`window._slSurgicallyUpdateArea = surgicallyUpdateArea;`。其他文件只需在需要时“从天上（window）取用”即可，完美解耦。

## 三、 重构轨迹：修改了哪些文件？

基于用户在实机测试中反馈的“闪烁”与“刷新不到位”问题，我们对以下文件进行了微创手术改造：

### 1. 输出模块区 (`module_output.js`)

- **问题**：点击历史记录缩略图、键盘方向键切换缩略图时，主视频会闪动。
- **解决**：将点击事件底部的 `saveAndRender()` 全部替换为调用 `applySurgicalUpdate(area)`。当只改变一张封面的数据时，只对该模块进行局部更新。

### 2. 输入模块区 (`module_input.js` & `ui_utils.js`)

- **问题**：在输入模块修改下拉框、布尔值、或上传图片时，会导致旁边毫不相干的输出视频发生闪烁。
- **解决**：在 `module_input.js` 中的各类 `change` 和 `click` 事件中移除 `saveAndRender()`，替换为局部爆破。特别地，在 `ui_utils.js` 的 `bindComboSelectEvents`（下拉框封装）中，加入回调钩子，一旦值发生变化，仅局部刷新被点击的那个输入模块。

### 3. 工具栏与状态联动 (`action_module_config.js`)

- **问题**：工具栏中的“关联节点”下拉框在选中后没有反应、“匹配媒体比例”开关切换时输入框不可用状态没有实时更新。
- **解决**：因为下方的模块变成了“局部刷新”，导致工具栏错过了“被一起重新生成”的机会。所以我们在工具栏所有操作事件的末尾，增加了**工具栏的自我刷新代码** (`renderDynamicToolbar` + `attachDynamicToolbarEvents`)，让它在不闪屏的情况下，自己更新自己按钮的高亮和数值状态。

### 4. 批量操作引擎 (`action_batch_sync.js`)

- **问题**：点击工具栏的“同步参数”、“前后移动”时，界面会发生核弹级全量闪烁。
- **解决**：
    - 对“同步参数”：找到匹配的模块后，对其 ID 逐个执行 `_slSurgicallyUpdateArea`。
    - 对“前后移动”：彻底放弃重新生成 HTML，而是利用原生 DOM 的 `appendChild()` 特性，将现有的 DOM 节点直接在父级列表中物理拽到新位置，实现“0 重绘”完美平移。

### 5. API 拦截与动态撑开 (`api_injector.js`)

- **问题**：勾选“匹配媒体比例”后，每次图片/视频生成完毕，出图的瞬间屏幕依然会闪一下。
- **解决**：原版在图片生成并在后台计算好高宽比例后，会广播一个全局重绘事件。我们将其拦截，在计算出准确的 `width` 和 `height` 后，同样只调用 `_slSurgicallyUpdateArea`，让该模块像动画一样自然地撑开到真实比例，绝不波及其他卡片的视频。
